Liste des Commandes Git

git init
Initialise un nouveau dépôt Git. Cela crée un nouveau sous-répertoire nommé .git qui contient tous les fichiers nécessaires au dépôt. Le répertoire de travail est considéré comme le "working directory".
git clone [url]
Clone un dépôt existant à partir de l'URL fournie. Cela crée une copie locale du dépôt distant, incluant toutes les branches et tags.
git add [fichier]
Ajoute un fichier à la zone de staging (index). Git ne suivra les modifications de ce fichier que si vous l'ajoutez à l'index.
git commit -m "message"
Enregistre les fichiers actuellement dans la zone de staging dans le dépôt, avec un message de commit expliquant la nature des modifications.
git status
Affiche l'état des fichiers dans le répertoire de travail et la zone de staging. Il montre quels fichiers ont été modifiés ou sont prêts à être commités.
git push [alias] [branche]
Pousse les commits de votre branche locale vers la branche distante. Alias est généralement origin, qui est le nom par défaut pour le dépôt distant que vous avez cloné.
git pull
Mise à jour de votre branche actuelle avec les derniers changements de la branche distante. Cela équivaut à faire un git fetch suivi d'un git merge.
git branch
Liste toutes les branches locales dans votre dépôt. Avec des options supplémentaires, vous pouvez également voir les branches distantes.
git checkout [nom_de_branche]
Permet de changer de branche ou de restaurer les fichiers de l'arborescence de travail.
git merge [branche]
Fusionne la branche spécifiée dans votre branche actuelle. Git essaiera de fusionner automatiquement les changements.
git stash
Mettez de côté les modifications non commitées de votre branche actuelle, vous permettant de passer à une autre branche sans commit.
